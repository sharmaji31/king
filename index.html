Considered Sorting

Sorting refers to the process of arranging data in a particular order such as ascending order or descending. An ascending sort of five integers {20,10,40,25,35} produces {10,20,25,35,40}. The main objective of sorting is to organize data so that it is more meaningful. Had the data not been sorted,we would spend hours to find a single piece of information.

The method of sorting can be classified as

 .Internal sorting
.External sorting

In internal sorting,all of the data which is to stored can be held in primary memory during sorting algorithm.

In external sorting,all of the data which is to be stored cannot be held in the primary memory at one time

There are four different types of sorting

1.Selection Sort

Simple sort is among the most instuitive of all the sort and works very fast for small amount of data. The basic idea of selection sort is that we repeatedly find the next smallest element in the list of elements and move it to its final position that it will occupy in the sorted list.



We can do this by swapping the elements at the first position with the smallest element. Now the smallest element is sorted and is placed at its proper position and need not be condidered again. Then find the next smallest element from the list and move to the correct position. This process continues until all the elements are moved to their correct position.

2.Bubble Sort

Bubble sort is one of the easiest techniques for sorting small arrays. It works by representing moving the largest elements to the highest index position in the array. For this,it uses several passes through the array and in each pass, the largest elements searches its proper position in the sorted array.



In Bubble sort, the first pass starts by comparing first element with second element and swapp if fist element is greater than second element. We then move to the next higher position element and repeat this process. This process of comparing and inter-changing is repeat until the largest element is bubbled from its original position to the correct position in the final sorted array

3.Insertion sort

Insertion sort is one of the simplest sorting algorithm. It works like the approach that a teacher might follow while answer sheet collected randomly from the students according to their roll number. The teacher begin sorting by assumping that the first answer sheet is correctly positioned. Then the second answer sheet is considered. If that answer sheet is out of order, it is placed before the first answer sheet. Then the teacher considered second and third answer sheet and place it at its proper position.



Formally,insertion sort takes one element at a time from the unsorted list and inserts it into a sorted list by scanning the list of sorted elements to determine where the new element will be placed.

What is stack ?

Stack is a linear data structure in which all insertion and deletion operation are restricted to only one end called top of the stack. In other words all elements are inserted and deleted only from top of the stack. For this reason, stack is said to have a Last In First Out (LIFO). Stack is logically a last in first out structure but physically it is implementation of array and linked list. Stack is not a real structure, but it just define how data is accessed rather than how it is store.

Operations of stack

 1.Push operation

Push operation is used to insert element on the top of the stack. After each push operation, the new inserted element become the top. Each push operation increase the number of element one by one.

2.Pop operation

Pop operation is used to delete elements from the top of the stack. After each pop operation, the top is decreased by one. If we perform pop operation on empty stack, stack is come in underflow state.

3.peek operation

Peek opration is used to display the elements which is placed on the top of the stack. Peek operation display the element without removing the element. After peek operation, the stack remain the same.

Circular queue

We observe that when we perform enqueue operation and dequeue operation the value of rear and front value is increased by one. But the main problem occur when rear variable reach the end of the array. suppose if there two locations are empty at begining of the array we can't insert element at that locations because the rear variable cannot cross the maxsize of the array. so a most efficient alternative is to let a queue that wrap around the array, such queue is known as circular queue.





In circular queue, last element is followed by fist element of the array. when the rear and front variable reach the maxsize of the array. Rear and front value is set to 1 and 0 respectively.

What is array.

Array is defined as a finite order set of homogenous elements.By the term homogenous we mean that there is a specific number of elements in an array which may be large and small but it may exist. By the term order we mean that all elements of the array are arranged in sequence. The homogenous property of array means that all the elements of the array are of same type.



So an array contain of all integers and all character but not the matrix of both.

Operations that we can perform on array

 1.Insert operation

Insert operation is refers to the process of inserting a new element in the array.



An element can be inserted either at any arbitraray position specified within the array if the array is sorted or at the end of the array if array is not sorted.

After inserting an element, the size of the array increase by 1. So before performing insert operation, one should make sure that there must be adequate memory space so as to accommodate the new element.

2.Delete operation

Delete refers to a process of removing an existing element from an array. In order to delete an element, you must frist search it using one of the technique i.e linear or binary searching. After searching the element it is removed and each subsequent is shifted one location upward so as to keep the order of array undisturbed.

Linked List

Singly linked list orsimply called linked list is a linear data structure consisting of elements called nodes where each node is composed of ywo parts : information part and a link part also called the next pointer part. The information part contains user supplied data and the link part which is a pointer that points tob the next node in the linked list . The link part of the last node is set to NULL that maeks thev end of the node.

Types Of Linked List
1.Circular linked list

the main drawback of singly linked list is that from a given node,we can access all the nodes that follow it but not the one preceding it. To overcome this problem, we make slight modifications in the singly linked list by replacing the NULL pointer in the last node of the list with the address of the first node. Such a list is called a singly linked circular list.


2.Bubbly Linked List

A bubbly linked list is a linear collection of elements called nodes such that each node consists of three parts :
1)An information part INFO which contains user supplied data.
2)A pointer part FLINK that points to the next node.
3)A pointer part PLINK that points to the previous node.

TREES

In computer science, a tree is a widely used abstract data type that represents a hierarchical tree structure with a set of connected nodes. Each node in the tree can be connected to many children (depending on the type of tree), but must be connected to exactly one parent,[1] except for the root node, which has no parent (i.e., the root node as the top-most node in the tree hierarchy).

These constraints mean there are no cycles or "loops" (no node can be its own ancestor), and also that each child can be treated like the root node of its own subtree, making recursion a useful technique for tree traversal. In contrast to linear data structures, many trees cannot be represented by relationships between neighboring nodes (parent and children nodes of a node under consideration if they exists) in a single straight line (called edge or link between two adjacent nodes).
